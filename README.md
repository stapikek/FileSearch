# FileSearch

**FileSearch** — приложение для Windows, которое индексирует файлы на локальных и сетевых дисках и позволяет быстро находить их по имени, размеру и другим параметрам.

Сайт проекта: [https://filesearch.pw/](https://filesearch.pw/)
Форум: [https://forum.filesearch.pw/](https://forum.filesearch.pw/)

---

## Возможности

- **Быстрый поиск по имени** — подстрока, маски (`*.txt`), точное совпадение по расширению (`report.pdf`)
- **Поиск с OR** — несколько условий через `|`, например: `Aurora | Xone`
- **Поиск по размеру** — синтаксис в стиле Everything: `size:19207kb`, `size:>10mb`
- **Фильтры в интерфейсе** — размер, дата изменения, тип файла
- **Подсветка совпадений** в таблице результатов
- **Контекстное меню** — открыть папку, перейти к файлу, копировать путь или имя
- **Индексация всех доступных дисков** — NTFS, FAT32, exFAT, ReFS, сетевые шары и др.
- **SQLite-база индекса** — данные сохраняются между запусками
- **Автоматическая переиндексация** при первом запуске, если база пуста
- **Системный трей** — работа в фоне, быстрый доступ к переиндексации
- **Тёмная тема** интерфейса
- **Два языка** — русский (по умолчанию) и английский
- **Проверка файловых систем** — встроенная диагностика томов

## FileSearch и Everything: в чём разница

[Everything](https://www.voidtools.com/) — эталонный инструмент мгновенного поиска файлов под Windows. **FileSearch** решает похожую задачу, но другим способом и с другим набором возможностей.

| | **FileSearch** | **Everything** |
|---|---|---|
| **Принцип индексации** | Полный обход файловой системы + SQLite | Чтение MFT/USN Journal на NTFS — практически мгновенно |
| **Скорость первой индексации** | Минуты на больших дисках | Секунды на NTFS |
| **Файловые системы** | NTFS, FAT32, exFAT, ReFS, UDF, CDFS, сетевые диски | Лучше всего NTFS; на других ФС — медленнее или ограниченно |
| **Потребление RAM** | Загружает индекс в память для поиска | Очень низкое за счёт MFT |
| **Синтаксис запросов** | Базовый: подстрока, `*`, `\|`, `size:` | Расширенный: `path:`, `dm:`, `dc:`, `ext:`, regex и др. |
| **Поиск по содержимому** | Нет | Нет (только имена и метаданные) |
| **HTTP/FTP/ETP сервер** | Нет | Есть |
| **Командная строка / SDK** | Только `--check-filesystems` | Полноценный CLI и SDK |
| **Портативность** | Требует Qt DLL (~десятки МБ) | Один маленький `.exe` |
| **Интерфейс** | Qt, тёмная тема, фильтры в UI | Минималистичный нативный UI |
| **Языки** | Русский + English | Много языков |
| **Лицензия** | MIT-подобная (см. «Лицензия» в программе) | Проприетарная (бесплатна для личного использования) |
| **Исходный код** | Открыт | Закрыт |

### Когда выбрать FileSearch

- Нужен поиск **не только на NTFS**, но и на **FAT32/exFAT-флешках**, ReFS, сетевых папках
- Важен **понятный GUI с фильтрами** и русским интерфейсом
- Нужен **открытый исходный код**, который можно собрать и изменить самостоятельно
- Достаточно базового синтаксиса поиска (имя, маска, OR, размер)

### Когда лучше Everything

- Нужен **максимально быстрый** поиск на NTFS с миллионами файлов
- Нужны **продвинутые операторы** (`path:`, `dm:`, `du:` и т.д.)
- Важны **CLI, SDK, HTTP-сервер** для интеграции с другими программами
- Нужен **минимальный размер** и потребление ресурсов

## Системные требования

- **ОС:** Windows 10 (1809+) / Windows 11 (Qt 6 официально не поддерживает Windows 7)
- **RAM:** от 2 ГБ (зависит от размера индекса)
- **Место на диске:** SQLite-база в `%AppData%\FileSearch\FileSearch.db`

---

## Сборка

### Зависимости

- [Qt 6.10+](https://www.qt.io/) (MinGW 64-bit): Core, Gui, Widgets, Sql, Svg
- [CMake](https://cmake.org/) 3.16+
- MinGW 13 (идёт в комплекте с Qt)

### Команды

```powershell
$env:PATH = "C:\Qt\6.10.0\mingw_64\bin;C:\Qt\Tools\mingw1310_64\bin;C:\Program Files\CMake\bin;$env:PATH"

cmake -S . -B build -G "MinGW Makefiles"
cmake --build build -j 8
```

Или запустите `build.bat`, затем `deploy.bat` для копирования Qt DLL в папку `build\`.

## Установщик

Кастомный мастер установки (Inno Setup): язык → лицензия → папка → данные → ярлыки → установка.

```bat
installer\build_installer.bat
```

Результат: `installer\output\FileSearch-Setup-1.0.0.exe` (включает Qt DLL и все зависимости).

Требуется [Inno Setup 6](https://jrsoftware.org/isdl.php).

## Запуск

```powershell
.\build\FileSearch.exe
```

При первом запуске программа автоматически начнёт индексацию, если база пуста. Индекс хранится в:

```
%AppData%\FileSearch\FileSearch.db
```

## Синтаксис поиска

| Запрос | Описание |
|--------|----------|
| `отчёт` | Поиск подстроки в имени файла |
| `*.docx` | Маска (wildcard) |
| `photo.jpg` | Точное совпадение имени с расширением |
| `Aurora \| Xone` | OR — файлы, подходящие под любой из терминов |
| `size:19207kb` | Точный размер (B, KB, MB, GB) |
| `size:>10mb` | Размер больше 10 МБ |

Дополнительно доступны фильтры **Размер**, **Дата** и **Тип** в панели под строкой поиска.

---

## Поддерживаемые файловые системы

NTFS, FAT, FAT32, exFAT, ReFS, UDF, CDFS, сетевые шары (SMB), RAM-диски.

Проверить тома на совместимость:

```powershell
.\build\FileSearch.exe --check-filesystems
```

или `check-fs.bat`.
